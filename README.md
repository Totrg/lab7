# lab7

```bash
---Docker---
`docker run` - запуск контейнера из образа Docker
`docker build` - создание нового образа Docker из Dockerfile
`docker images` - список всех образов Docker, которые установлены на хосте
`docker ps` - список всех запущенных контейнеров
`docker stop` - остановка запущенного контейнера
`docker start` - запуск остановленного контейнера
`docker restart` - перезапуск контейнера
`docker rm` - удаление контейнера
`docker rmi` - удаление образа
`docker exec` - выполнение команды внутри запущенного контейнера
`docker pull` - загрузка образа Docker из репозитория Docker Hub или другого реестра образов
`docker push` - отправка локального образа Docker в репозиторий Docker Hub или другой реестр образов
```
```bash
---Dockerfile---
`FROM` - указывает базовый образ, от которого наследуется новый образ. Обычно это минимальный образ ОС, например, Ubuntu, Debian, Alpine или другой.
`WORKDIR` - это инструкция в Dockerfile, которая устанавливает рабочую директорию для любых команд, которые будут выполняться внутри контейнера
`RUN` - выполняет команду внутри контейнера в процессе создания образа.
`COPY` - копирует файлы или директории из хост-системы в контейнер.
`CMD` - определяет команду, которая будет запущена внутри контейнера при его запуске.
```
